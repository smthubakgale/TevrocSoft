Building a Full Firebase JSON Editor System

Real-time collaboration, file management, authentication, live sync, presence tracking, and role-based access control

1. Introduction

This project is a fully functional cloud-based JSON editor system built using Firebase and vanilla JavaScript. It behaves like a simplified version of Google Docs but specifically designed for structured JSON data.

The system includes:

  • Google authentication login system
  • Real-time database sync using Firebase Realtime Database
  • ACE code editor for JSON editing
  • File creation, renaming, deletion
  • Multi-user collaboration system
  • Live presence tracking
  • Role-based access (admin, editor, viewer)
This system is designed for developers building SaaS tools, admin dashboards, or collaborative document systems.

2. System Architecture Overview

The system is structured around Firebase as the backend: 


Frontend (HTML + JS + ACE Editor)
        |
        v
Firebase Authentication
        |
        v
Firebase Realtime Database
        |
        v
Live Sync + Collaboration + File Storage

There is no traditional backend server. Firebase handles everything.

3. Firebase Setup Explained

We initialize Firebase like this: 


const firebaseConfig = {
  apiKey: "...",
  authDomain: "app.firebaseapp.com",
  databaseURL: "https://app.firebaseio.com",
  projectId: "app-id"
};

firebase.initializeApp(firebaseConfig);

const auth = firebase.auth();
const db = firebase.database();

This connects:

  • Authentication service
  • Realtime database

4. Authentication Flow (Google Login)

Users log in using Google OAuth popup: 


loginBtn.onclick = () => {
  const provider = new firebase.auth.GoogleAuthProvider();
  auth.signInWithPopup(provider);
};

We listen for login state changes: 


auth.onAuthStateChanged(user => {
  if (user) {
    console.log("Logged in:", user.displayName);
  }
});

What happens after login?

  • User profile is stored in Firebase
  • Profile picture is displayed
  • User is mapped to UID
  • File access is loaded

5. User Profile System

We store user metadata like this: 


db.ref("users/" + user.uid).set({
  email: user.email,
  displayName: user.displayName,
  photoURL: user.photoURL
});

This allows:

  • Displaying profile pictures in UI
  • Mapping emails to user IDs
  • Collaboration identification

6. File System Structure

Files are structured like this in Firebase: 


files/
  userUID/
    filename/
      owner: userUID
      content: {...JSON...}
      collaborators: {...}

This structure ensures:

  • Each user owns their files
  • Files can be shared across users
  • Permissions are stored per file

7. Creating a New File


db.ref(`files/${currentUser.uid}/${filename}`).set({
  owner: currentUser.uid,
  content: jsonData,
  collaborators: {
    [currentUser.uid]: "admin"
  }
});
Every new file automatically assigns the creator as "admin".

8. ACE Editor Integration

The ACE editor provides syntax highlighting and JSON formatting: 


const editor = ace.edit("editor");
editor.setTheme("ace/theme/monokai");
editor.session.setMode("ace/mode/json");
editor.setOption("fontSize", "14px");

Why ACE editor?

  • Lightweight and fast
  • Supports syntax highlighting
  • Great for structured data

9. Real-Time Sync Engine (Core Feature)

This is the most important part of the system.


liveSyncRef = db.ref(`${currentFilePath}/content`);

liveSyncRef.on("value", snap => {
  const data = snap.val();
  editor.setValue(JSON.stringify(data, null, 2));
});

How it works:

  • Any change in Firebase triggers update
  • All users viewing file see changes instantly
  • No manual refresh required
If not handled carefully, this can cause infinite update loops. Cursor preservation logic is needed.

10. Presence System (Online Users)

We track who is currently viewing a file: 


const presenceRef = db.ref(`presence/${fileRoomId}/${currentUser.uid}`);

presenceRef.set({
  uid: currentUser.uid,
  displayName: currentUser.displayName,
  photoURL: currentUser.photoURL
});

presenceRef.onDisconnect().remove();

This enables:

  • Live user list
  • “Online users” indicator
  • Collaboration awareness

11. Collaboration System

Each file contains collaborators with roles: 


collaborators: {
  uid1: "admin",
  uid2: "editor",
  uid3: "viewer"
}

Roles control permissions:

  • Admin – full control (edit, delete, invite)
  • Editor – can modify content
  • Viewer – read-only access

12. Invitation System

Users can invite others by email: 


inviteBtn.onclick = () => {
  const emailKey = email.replace(/\./g, "_");

  db.ref("usersByEmail/" + emailKey).once("value", snap => {
    let uid = snap.val();

    if (!uid) {
      uid = db.ref("users").push().key;
      db.ref("users/" + uid).set({ email });
      db.ref("usersByEmail/" + emailKey).set(uid);
    }

    db.ref(currentFilePath + "/collaborators/" + uid)
      .set(role);
  });
};

This allows collaboration even for users not yet registered.

13. File Renaming System


if (newFilename !== originalFilename) {
  userRef.child(newFilename).set(data);
  userRef.child(originalFilename).remove();
}

Renaming is implemented as:

  • Create new node
  • Copy data
  • Delete old node

14. File Deletion Strategy

Instead of immediate deletion:


fileRef.child("meta").set({
  deleted: true
});

Then after delay:


fileRef.remove();

Why?

  • Prevents sync crashes
  • Lets collaborators receive update
  • Ensures UI cleanup

15. Uploading JSON Files


reader.onload = e => {

Supports importing external JSON files directly.

16. Email-to-UID Mapping

We map emails to Firebase IDs: 


function encodeEmail(email){
  return email.replace(/\./g,"_");
}

This allows:

  • Fast lookup
  • Invitation system
  • User resolution without login

17. Performance Considerations

  • Use on() carefully to avoid memory leaks
  • Detach listeners when switching files
  • Limit Firebase writes per second
  • Debounce editor changes (recommended upgrade)

18. Security Considerations

Firebase rules are essential. Without them, anyone can access or modify data.

Recommended rules concept:


{
  "rules": {
    "files": {
      "$uid": {
        ".read": "auth != null",
        ".write": "auth != null && auth.uid == $uid"
      }
    }
  }
}
